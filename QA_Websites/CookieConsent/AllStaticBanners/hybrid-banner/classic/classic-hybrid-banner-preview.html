    üéØ What is Staging Mode?

    Staging Mode (also called Preview Mode) allows you to:

    • ‚úì Preview banner configurations before they go live in production
    • ‚úì Test new banner designs and settings without affecting end users
    • ‚úì Validate cookie categories and consent workflows
    • ‚úì Review banner text, translations, and customizations
    • ‚úì Ensure compliance requirements are met before deployment

    Important: Staging mode should only be used in testing/QA environments. Never enable staging mode in production as it may show unpublished or draft banner configurations.

    ‚úÖ Testing Instructions

    1. Initial Load: When the page loads, the cookie consent banner should appear with the STAGING configuration.
    2. Manage Preferences: Click the "Manage Cookie Preferences" button below to reopen the banner.
    3. View Consent UUID: Click the blue button in the top-right corner to see captured consent UUIDs.
    4. Preview Changes: Any unpublished changes in the CMP admin will be visible with staging mode enabled.
    5. Test Consent Actions: Accept, reject, or customize cookie preferences and verify the consent is recorded.
    6. Compare Versions: Test alongside the non-staging version to see differences.
    7. Verify Storage: Check browser's local storage and cookies to confirm consent data is saved correctly.

    üìä Expected Behavior
