    üéØ Multi-Level Banner Features

    Multi-level banners provide enhanced granularity for cookie consent management:

    • Hierarchical Consent: Users can drill down into different levels of consent options
    • Vendor-Level Control: Manage consent for individual vendors/partners
    • Purpose-Based Consent: Control consent based on specific purposes (analytics, marketing, etc.)
    • Cookie-Level Granularity: View and manage individual cookies
    • IAB TCF Support: Compatible with IAB Transparency & Consent Framework
    • Enhanced Transparency: Provides detailed information about data processing

    üéØ What is Staging Mode?

    Staging Mode (also called Preview Mode) allows you to:

    • ‚úì Preview multi-level banner configurations before they go live in production
    • ‚úì Test new vendor lists, purposes, and consent hierarchies without affecting end users
    • ‚úì Validate IAB TCF configurations and vendor signals
    • ‚úì Review detailed vendor information, purposes, and legitimate interest claims
    • ‚úì Test multi-level navigation and user experience flows
    • ‚úì Ensure compliance requirements are met before deployment

    Important: Staging mode should only be used in testing/QA environments. Never enable staging mode in production as it may show unpublished or draft banner configurations.

    ‚úÖ Testing Instructions

    1. Initial Load: When the page loads, the multi-level consent banner should appear with the STAGING configuration.
    2. Explore Hierarchy: Navigate through different levels of consent options (categories ‚Üí vendors ‚Üí purposes ‚Üí cookies).

    8. Verify IAB Compliance: Check that IAB TC strings are generated correctly with draft vendor lists.
    9. Verify Storage: Check browser's local storage and cookies to confirm detailed consent data is saved.
